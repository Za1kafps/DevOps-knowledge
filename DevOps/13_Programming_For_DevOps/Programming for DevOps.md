# Programming for DevOps

Программирование в DevOps нужно не ради написания продуктового backend, а для автоматизации повторяемых операций, интеграции API, проверки конфигурации и создания эксплуатационных инструментов.

## Что нужно уметь

- написать идемпотентный [[Bash]]-скрипт для локальной операции;
- обработать JSON/YAML и вызвать HTTP API через [[Python]];
- собрать статический CLI или exporter на [[Go]];
- определить контракт входных данных и корректные [[Exit Codes]];
- покрыть критическую логику тестами;
- добавить timeout, retry с backoff и структурированные логи;
- не передавать секреты через argv, логи и небезопасные временные файлы.

## Выбор инструмента

`Bash`:

- короткая последовательность системных команд;
- glue-код в CI;
- работа с процессами, файлами и потоками.

`Python`:

- API-клиенты;
- обработка JSON/YAML;
- сложная логика и быстрое прототипирование;
- автоматизация, где важны библиотеки и читаемость.

`Go`:

- переносимый бинарник без runtime на целевой машине;
- параллельные network operations;
- CLI, controller, webhook, exporter или долгоживущий daemon.

Если shell-скрипт разросся до сложных структур данных, нескольких уровней retry и большого количества ветвлений, его пора переносить на Python или Go.

## Инженерные свойства автоматизации

### Идемпотентность

Повторный запуск приводит систему к тому же желаемому состоянию:

```text
ensure user exists
ensure directory has mode 0750
ensure configuration contains value
```

Команда `create` без проверки состояния обычно неидемпотентна. Операция `ensure` сначала читает состояние, затем меняет только необходимое.

### Наблюдаемость

Инструмент должен сообщать:

- какая операция выполняется;
- над каким объектом;
- сколько заняла;
- чем завершилась;
- какой correlation/request ID вернул API.

Секреты, токены и полные тела чувствительных запросов логировать нельзя.

### Ограниченное ожидание

У каждого внешнего вызова должен быть timeout. Retry допустим только для временных и безопасно повторяемых ошибок. Без backoff и jitter несколько клиентов создают retry storm.

### Предсказуемое завершение

- `0` — операция успешна;
- ненулевой код — ошибка;
- `stdout` — полезный результат;
- `stderr` — диагностика;
- `SIGTERM` обрабатывается для корректного завершения.

## Проверки перед merge

```bash
shellcheck scripts/*.sh
python -m compileall .
python -m pytest
gofmt -w .
go vet ./...
go test -race ./...
yamllint .
```
