# lsof

## Что это

`lsof` — показывает открытые файлы у процессов.

В [[Linux]] файл — это не только обычный файл, но и сокет, порт, pipe, device.

## Зачем нужен

`lsof` помогает понять:

какой процесс держит файл  
какой процесс слушает порт  
какие файлы открыл процесс  
почему диск не размонтируется  
какой процесс использует deleted-файл

## Команды
```bash
lsof -p PID` — файлы конкретного процесса  
lsof -i` — сетевые соединения  
lsof -i :80` — кто использует порт 80  
lsof -i TCP:443` — TCP-порт 443  
lsof /path/file` — кто держит файл  
lsof +D /path/dir` — кто использует директорию
```


## Частые кейсы

Кто занял порт:

`sudo lsof -i :8080`

Кто держит файл:

`lsof /var/log/app.log`

Кто держит удаленный файл:

`lsof | grep deleted`

Если deleted-файл открыт процессом, место на диске может не освобождаться до перезапуска процесса.
