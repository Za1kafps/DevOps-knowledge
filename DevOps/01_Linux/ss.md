# ss

## Что это

`ss` — утилита для просмотра [[Network]] сокетов в [[Linux]].

Сокет — это точка сетевого соединения. Через него процесс слушает порт или подключается к другому хосту.

`ss` часто используют вместо `netstat`, потому что он быстрее и берет данные напрямую из ядра.
## Зачем нужен

`ss` помогает понять:

какой порт слушается  
какой процесс держит порт  
есть ли активные TCP/UDP соединения  
в каком состоянии TCP-соединение  
куда подключается приложение  
есть ли проблемы с сетью на уровне сокетов

## Внутри как работает

`ss` читает информацию о сокетах из ядра Linux.

Основные источники:

`/proc/net/tcp`  
`/proc/net/udp`  
`/proc/net/unix`  
Netlink-интерфейс ядра

То есть `ss` показывает не “мнение программы”, а состояние сетевых соединений в ядре.

## Основные команды

```
ss -tulpen
ss -lntp
ss -tunap
ss -ant
ss -uap
ss -xap
ss -s
```

## Расшифровка флагов

```
-t  TCP
-u  UDP
-l  listening sockets
-n  не резолвить имена, показывать IP и порты числами
-p  показать процесс
-a  все сокеты
-e  расширенная информация
-s  статистика
-x  Unix-сокеты
```

## Полезные команды

Показать TCP/UDP порты, которые слушаются:

```
sudo ss -tulpen
```

Показать только TCP listening-порты:

```
sudo ss -lntp
```

Кто слушает порт 80:

```
sudo ss -lntp | grep ':80'
```

Все TCP-соединения:

```
ss -ant
```

Все UDP-сокеты:

```
sudo ss -uap
```

Все соединения конкретного процесса:

```
sudo ss -tunap | grep nginx
```

Статистика по сокетам:

```
ss -s
```

Unix-сокеты:

```
ss -xap
```

## TCP states

```
LISTEN       процесс слушает порт
ESTAB        соединение установлено
SYN-SENT     клиент отправил SYN
SYN-RECV     сервер получил SYN
TIME-WAIT    соединение закрыто, но ядро еще держит запись
CLOSE-WAIT   удаленная сторона закрылась, локальная еще нет
FIN-WAIT     идет закрытие соединения
```

## Частые кейсы

Проверить, слушает ли приложение порт:

```
sudo ss -lntp | grep ':3000'
```

Проверить, кто занял порт:

```
sudo ss -lntp | grep ':8080'
```

Проверить много ли соединений:

```
ss -ant | wc -l
```

Проверить соединения в TIME-WAIT:

```
ss -ant | grep TIME-WAIT | wc -l
```

Проверить соединения к конкретному IP:

```
ss -ant | grep '1.2.3.4'
```

## Плюсы

быстрее netstat  
есть почти везде  
показывает процессы через `-p`  
удобен для troubleshooting сети  
хорошо подходит для серверов и контейнеров

## Минусы

без `sudo` не всегда показывает процесс  
вывод может быть непривычный  
не показывает весь путь пакета  
не заменяет tcpdump  
не показывает DNS-проблемы напрямую

## Альтернативы


## Важные недочеты

Если порт слушается на `127.0.0.1`, он доступен только локально.

Пример:

```
127.0.0.1:3000
```

Если порт слушается на `0.0.0.0`, он слушается на всех IPv4-интерфейсах.

Пример:

```
0.0.0.0:80
```

Если порт слушается на `[::]`, это IPv6. Иногда он может принимать и IPv4, зависит от настроек ядра.
