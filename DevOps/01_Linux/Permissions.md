# Permissions
Permissions — права доступа к файлам и директориям в [[Linux]].

Права определяют, кто может читать, изменять и запускать файл.
## Базовые права

r — read, читать  
w — write, писать  
x — execute, выполнять

Права задаются для:

user — владелец  
group — группа  
others — остальные

## Пример

`-rwxr-xr--`

Расшифровка:

user: rwx  
group: r-x  
others: r--

## Команды
```bash
ls -l — посмотреть права  
chmod +x script.sh — добавить запуск  
chmod 644 file.txt — обычный файл  
chmod 755 script.sh — скрипт/директория  
chown user:user file.txt — сменить владельца  
chown -R user:user /path — сменить владельца рекурсивно
```

## Частые значения

644 — обычный файл  
755 — директория или скрипт  
600 — приватный файл  
700 — приватная директория  
777 — всем всё можно, обычно плохая практика
## Права директорий

Для директории `x` означает право зайти внутрь.

Если нет `x` на директории, файл внутри может существовать, но доступа к нему не будет.

## Проверка проблем
```bash
ls -l file
namei -l /path/to/file
id
groups
```
## Частые причины Permission denied

нет прав на файл  
нет прав на директорию выше  
не тот владелец  
файл принадлежит root  
скрипт без execute  
сервис запущен от другого пользователя  
файловая система смонтирована read-only
