# DELETE

`DELETE` — HTTP-метод для удаления ресурса.

Пример:

```text
DELETE /users/42
```

---

# Проверка

```bash
curl -v -X DELETE https://example.com/api/users/42 \
  -H "Authorization: Bearer TOKEN"
```

---

# Ответы

Частые успешные ответы:

```text
200 OK
202 Accepted
204 No Content
```

`202` означает, что удаление принято, но может выполняться асинхронно.

`204` означает успешный ответ без body.

---

# Idempotency

DELETE обычно проектируют idempotent.

Повторный DELETE того же ресурса не должен создавать новое побочное действие.

Но ответ может отличаться:

```text
первый DELETE -> 204
второй DELETE -> 404
```

Это зависит от API.

---

# Частые ошибки

## 401/403

Удаление запрещено.

## 404

Ресурс уже удалён или указан неправильный path.

## 409

Ресурс нельзя удалить из-за зависимостей.
