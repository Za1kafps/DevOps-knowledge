# IPv6

`IPv6` — версия IP с 128-битными адресами.

Пример адреса:

```text
2a00:1450:4001:82a::200e
```

IPv6 решает проблему нехватки IPv4-адресов, но в DevOps чаще всего ломает ожидания: сервис слушает IPv6, DNS отдаёт AAAA-запись, firewall настроен только для IPv4.

---

# Главное отличие от IPv4

В IPv6 адресов достаточно, поэтому NAT не является обязательной частью архитектуры.

Обычная идея:

```text
у хоста может быть публичный IPv6
маршрутизация работает напрямую
firewall становится особенно важен
```

Это отличается от IPv4, где частные сети и [[NAT]] встречаются почти везде.

---

# Виды адресов

## Loopback

```text
::1
```

IPv6-аналог `127.0.0.1`.

---

## Link-local

```text
fe80::/10
```

Адреса внутри одного L2-сегмента.

Часто видны на интерфейсах автоматически:

```bash
ip -6 addr
```

---

## Global unicast

Публичные маршрутизируемые IPv6-адреса.

Именно они используются для доступа из интернета.

---

# DNS и IPv6

IPv6-адрес домена хранится в AAAA-записи:

```bash
dig example.com AAAA
```

Если у домена есть `AAAA`, клиент может попробовать IPv6 раньше IPv4.

Симптом:

```text
у части пользователей сайт открывается медленно или не открывается
```

Причина может быть в сломанном IPv6 path, хотя IPv4 работает.

---

# Проверка

Показать IPv6-адреса:

```bash
ip -6 addr
```

Показать IPv6 routes:

```bash
ip -6 route
```

Проверить IPv6 DNS:

```bash
dig AAAA example.com
```

Проверить HTTP по IPv6:

```bash
curl -6 -v https://example.com
```

Проверить HTTP только по IPv4:

```bash
curl -4 -v https://example.com
```

# Частые проблемы

## Сервис слушает `[::]`

Пример:

```text
[::]:8080
```

Это IPv6 wildcard. В Linux поведение с IPv4 зависит от `net.ipv6.bindv6only`.

Проверить:

```bash
sysctl net.ipv6.bindv6only
ss -lntp
```

---

## Firewall настроен только для IPv4

Можно закрыть порт в `iptables`, но забыть про `ip6tables`/`nftables`.

Проверка:

```bash
sudo nft list ruleset
sudo ip6tables -S
```

---

## AAAA-запись есть, а IPv6 на сервере не готов

Если DNS отдаёт IPv6, но сервер или балансировщик по нему не работает, часть клиентов будет страдать.

Проверить:

```bash
dig AAAA example.com
curl -6 -v https://example.com
```
