# Subnetting and CIDR

`CIDR` — способ записывать IP-сеть через префикс.

Пример:

```text
192.168.1.0/24
```

`/24` означает, что первые 24 бита — сеть, остальные 8 бит — адреса хостов.

Тема относится к [[IPv4]], [[IPv6]], [[Routing]], VPC, Kubernetes Pod CIDR и Docker networks.

---

# Зачем это нужно

CIDR отвечает на вопрос:

```text
какие IP считаются локальными для этой сети?
```

От этого зависит:

```text
куда хост отправит пакет
какой gateway будет использован
сколько адресов доступно
пересекаются ли сети
можно ли соединить сети через VPN
```

---

# Пример `/24`

```text
network:   192.168.1.0/24
mask:      255.255.255.0
range:     192.168.1.0 - 192.168.1.255
hosts:     192.168.1.1 - 192.168.1.254
broadcast: 192.168.1.255
```

В классической IPv4-сети первый адрес — network, последний — broadcast.

---

# Частые маски

```text
/32  один IPv4-адрес
/30  маленькая point-to-point сеть
/24  256 адресов
/20  4096 адресов
/16  65536 адресов
/8   очень большая сеть
```

Чем меньше число после `/`, тем больше сеть.

---

# Проверка на Linux

Показать адреса и маски:

```bash
ip addr
```

Пример:

```text
inet 10.0.1.15/24
```

Это значит:

```text
IP: 10.0.1.15
network: 10.0.1.0/24
```

Проверить, куда пойдёт пакет:

```bash
ip route get 10.0.2.20
```

# Пересечение сетей

Самая неприятная ошибка в VPN, VPC peering и Kubernetes — overlapping CIDR.

Пример:

```text
office: 10.0.0.0/16
cloud:  10.0.0.0/16
```

Маршрутизатор не может нормально понять, где находится `10.0.5.10`, потому что такая сеть есть с двух сторон.

Для Kubernetes важно заранее планировать:

```text
node subnet
pod CIDR
service CIDR
VPN CIDR
office CIDR
```

---

# Ошибки в DevOps

## Слишком маленькая подсеть

В Kubernetes или VPC быстро заканчиваются адреса.

Симптомы:

```text
новые nodes не создаются
pods не получают IP
load balancer не получает address
```

---

## Слишком большая подсеть

Труднее изолировать окружения и строить понятные routes.

---

## Неверный route из-за более специфичного префикса

Правило longest prefix match:

```text
10.0.1.0/24 важнее, чем 10.0.0.0/16
```

Если есть два маршрута, выбирается более точный.
