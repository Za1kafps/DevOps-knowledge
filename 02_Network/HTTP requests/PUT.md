# PUT

`PUT` — HTTP-метод для полной замены ресурса по известному адресу.

Пример:

```text
PUT /users/42
```

Идея: клиент отправляет полное новое состояние ресурса.

---

# PUT и POST

`POST` часто создаёт ресурс или запускает действие.

`PUT` обычно записывает ресурс по конкретному URI.

Пример:

```text
POST /users       создать пользователя, id выберет сервер
PUT /users/42     заменить пользователя 42
```

---

# PUT и PATCH

`PUT` — полная замена.

`PATCH` — частичное изменение.

Если отправить неполный объект в PUT, backend может удалить отсутствующие поля, если API так спроектирован.

---

# Проверка

```bash
curl -v -X PUT https://example.com/api/users/42 \
  -H "Content-Type: application/json" \
  -d '{"name":"Ivan","role":"admin"}'
```

---

# Частые ошибки

## 404

Ресурс не найден или API не создаёт ресурс через PUT.

## 409

Конфликт состояния, version mismatch, optimistic lock.

## 415

Неверный `Content-Type`.
