# Memory and OOM
## Что это

Память в [[Linux]] используется процессами, ядром, page cache и buffers.

Свободная память не всегда должна быть большой: Linux активно использует RAM под кэш.
## Основные понятия

RAM — физическая память  
Swap — место на диске как запасная память  
Page cache — кэш файлов в памяти  
RSS — реальная память процесса в RAM  
VSZ — виртуальная память процесса  
OOM — ситуация, когда памяти не хватает

## Команды
```bash
free -h — память системы  
top — процессы и память  
htop — удобный просмотр  
ps aux --sort=-%mem — процессы по памяти  
vmstat 1 — память и swap live  
cat /proc/meminfo — подробная информация
```

free — совсем свободная память  
available — память, которую реально можно использовать

Обычно смотреть надо на `available`, а не только на `free`.

## OOM Killer

Если памяти не хватает, ядро может убить процесс через OOM Killer.

Проверить:

`journalctl -k -b | grep -i oom`

или:

`dmesg -T | grep -i killed`

## Лимиты памяти

Лимиты процесса:

`cat /proc/PID/limits`

Лимиты shell:

`ulimit -a`

Если сервис под systemd, лимит может быть в unit-файле:

`systemctl cat service-name`

Искать параметры типа:

`MemoryMax`  
`TasksMax`  
`LimitNOFILE`

## Частые причины проблем

процесс течет по памяти  
слишком маленький swap  
много процессов  
сервис уперся в лимит  
слишком много открытых файлов  
ядро убило процесс через OOM
