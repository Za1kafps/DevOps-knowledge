# Linux Processes
## Что это

Процесс — запущенная программа в [[Linux]].
У каждого процесса есть:

- PID — id процесса  
- PPID — id родительского процесса  
- UID — пользователь, от которого запущен процесс  
- State — состояние процесса  
- FD — открытые файлы/сокеты  
- Limits — ограничения процесса

## Жизненный цикл

Процесс обычно создается через `fork()`, потом заменяет код через `exec()`.

Родительский процесс может ждать завершения дочернего через `wait()`.

Если родитель умер, процесс может перейти под управление `systemd`.

## Состояния процессов

`R — running, выполняется`  
`S — sleeping, ждет событие`
`D — uninterruptible sleep, часто ожидание диска/IO`  
`Z — zombie, процесс завершился, но родитель не забрал статус`  
`T — stopped, остановлен`

## Команды
```bash
`ps aux` — список процессов  
`ps -ef` — процессы с PPID  
`top` — процессы live  
`htop` — удобный top  
`pgrep nginx` — найти PID  
`pstree -p` — дерево процессов  
`kill PID` — попросить процесс завершиться  
`kill -9 PID` — принудительно убить
```

## Лимиты процесса

Посмотреть лимиты процесса:

`cat /proc/PID/limits`

Посмотреть лимиты текущей shell:

`ulimit -a`

Частые лимиты:

open files — сколько файлов/сокетов можно открыть  
max processes — сколько процессов можно создать  
max memory size — лимит памяти  
cpu time — лимит CPU-времени

## Полезно для диагностики

Если процесс завис:
```bash
ps -o pid,ppid,state,cmd -p PID
```
Если много процессов:
```bash
ps aux --sort=-%cpu
```

Если много памяти:
```bash
ps aux --sort=-%mem
```

Если непонятно, что держит процесс:
```bash
lsof -p PID
```
